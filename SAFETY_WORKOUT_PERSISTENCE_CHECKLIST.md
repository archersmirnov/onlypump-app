# Чеклист проверки сохранения тренировок

Дата: 2026-07-01

Цель: проверить не кодом, а реальным сценарием в приложении, что тренировки создаются, повторяются, редактируются и удаляются с сохранением в постоянной базе.

Этот файл не меняет поведение приложения.

## Когда проходить

После обновления `index.html` в той версии, которую реально открывает приложение.

Перед проверкой:

- обновить страницу приложения;
- открыть Supabase logs для `onlypump-workouts-api`;
- держать рядом таблицу `workouts`, `workout_exercises`, `workout_sets`.

## Минимальный набор сценариев

### 1. Создание обычной тренировки

1. Создать новую тренировку на сегодня.
2. Добавить 1 упражнение.
3. Добавить 2 подхода.
4. Поменять вес и повторы.
5. Обновить приложение.

Ожидаемо:

- тренировка осталась;
- упражнение осталось;
- подходы остались;
- вес и повторы не откатились;
- в logs нет красной ошибки на сохранение.

### 2. Повторить и начать сейчас

1. Взять старую тренировку.
2. Нажать повторить -> сейчас.
3. Удалить один подход.
4. Изменить другой подход.
5. Завершить тренировку.
6. Обновить приложение.

Ожидаемо:

- завершенная тренировка осталась completed;
- удаленный подход не вернулся;
- измененный подход сохранился;
- объем и подходы выглядят логично.

### 3. Повторить как выполненную

1. Взять старую тренировку.
2. Повторить как уже выполненную.
3. Обновить приложение.

Ожидаемо:

- тренировка есть в completed;
- упражнения и подходы есть;
- она не превращается в active/planned.

### 4. Редактирование completed-тренировки

1. Открыть уже завершенную тренировку.
2. Изменить название.
3. Удалить подход.
4. Добавить подход.
5. Поменять вес/повторы/RIR/RPE.
6. Обновить приложение.

Ожидаемо:

- название сохранилось;
- удаленный подход не вернулся;
- новый подход сохранился;
- изменения подходов сохранились.

### 5. Удаление

1. Создать тестовую тренировку.
2. Добавить упражнение и подход.
3. Удалить тренировку.
4. Обновить приложение.

Ожидаемо:

- тренировка не возвращается;
- связанные упражнения и подходы не отображаются;
- в logs нет ошибки удаления.

### 6. Программы и запланированные

1. Добавить программу в расписание.
2. Удалить одну запланированную тренировку только для себя.
3. Обновить приложение.

Ожидаемо:

- удаленная тренировка не вернулась;
- чужие пользователи не затронуты;
- остальные запланированные тренировки этой программы остались.

## Что считать стоп-сигналом

- Красная ошибка в logs `onlypump-workouts-api`.
- После reload возвращается удаленный подход.
- Completed-тренировка откатывает изменения.
- Повтор как выполненная не появляется после reload.
- Удаление тренировки визуально прошло, но после reload она снова появилась.

Если появился стоп-сигнал, дальше не рефакторить. Сначала смотреть конкретный action в logs и id тренировки/подхода, который откатился.
